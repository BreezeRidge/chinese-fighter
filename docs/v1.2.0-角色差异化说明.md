## 🎯 解决方案

### 核心思路
**使用调色板技术为每个角色应用不同颜色**
- 保持原精灵图的细节和质量
- 通过颜色体现门派特征
- 无需绘制新精灵图

### 技术选型：HSV色彩空间

**为什么选择HSV？**
1. **色调独立**：可单独调整颜色，不影响亮度
2. **直观控制**：色调偏移30°即可产生明显差异
3. **保持结构**：饱和度和明度保持原比例

**RGB vs HSV对比**：
```
RGB色彩空间：
  - 红色(255,0,0) → 绿色(0,255,0) 需要复杂插值
  - 难以保持原图的明暗关系

HSV色彩空间：
  - H(色调): 0-360° 的色环，30°偏移即可换色
  - S(饱和度): 0-1，控制颜色鲜艳度
  - V(明度): 0-1，控制亮暗
  - 可独立调整，互不影响
```

## 🛠️ 技术实现

### 1. 工具脚本开发

**文件**: `tools/generate_character_palettes_pure.py`

**核心功能**：
```python
def apply_color_palette(pixels, hue_shift, saturation_factor, value_factor):
    """应用调色板到像素数据"""
    for each pixel (r, g, b, a):
        # 1. 跳过透明像素
        if a == 0: continue
        
        # 2. 保护黑色轮廓线
        if r<30 and g<30 and b<30: continue
        
        # 3. 保护白色高光
        if r>225 and g>225 and b>225: continue
        
        # 4. RGB → HSV
        h, s, v = rgb_to_hsv(r, g, b)
        
        # 5. 应用变换
        h = (h + hue_shift) % 1.0
        s = min(1.0, s * saturation_factor)
        v = min(1.0, v * value_factor)
        
        # 6. HSV → RGB
        r_new, g_new, b_new = hsv_to_rgb(h, s, v)
```

**技术亮点**：
- ✅ 纯Python标准库实现（无外部依赖）
- ✅ PNG文件直接读写（struct + zlib）
- ✅ 像素级精确处理
- ✅ 保护轮廓线和高光

## 🎨 色彩理论

### HSV色彩空间详解

**色调环（Hue Wheel）**：
```
  0° - 红色
 30° - 橙色  ← 少林僧
 60° - 黄色
120° - 绿色  ← 武当道士
180° - 青色
200° - 青蓝  ← 峨眉剑客
240° - 蓝色
300° - 紫色
360° - 回到红色
```

**饱和度（Saturation）**：
- 0.0：灰色（无色彩）
- 0.5：柔和色彩
- 1.0：纯色（最鲜艳）
- >1.0：增强饱和（可能过于刺眼）

**明度（Value）**：
- 0.0：黑色
- 0.5：中等亮度
- 1.0：最亮
- >1.0：过曝（不推荐）

### 为什么保护轮廓线和高光？

**黑色轮廓线**（RGB < 30）：
- 作用：定义角色边界和细节
- 如果变色：轮廓会变成彩色，失去清晰度
- 保护策略：跳过HSV变换

**白色高光**（RGB > 225）：
- 作用：表现光照效果和立体感
- 如果变色：高光会变成彩色，失去闪亮效果
- 保护策略：跳过HSV变换
