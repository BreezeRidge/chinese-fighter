# v1.1.0 精灵图升级说明

## 📋 升级概述

**版本**: v1.1.0  
**日期**: 2026-07-13  
**主题**: 高质量角色精灵图升级  
**问题**: "游戏中人的人物/动作，不够真实"

---

## 🎯 问题诊断

### 原有精灵图的问题

1. **质量低**
   - 文件大小：仅7-8KB
   - 内容：简单色块占位符
   - 细节：无角色特征，纯色填充

2. **帧数少**
   - Idle：4帧
   - Walk：6帧
   - Jump：4帧
   - Attack Light：4帧
   - Attack Heavy：6帧
   - Hurt：3帧
   - **结果**：动作僵硬，不流畅

3. **动画速度不合理**
   - 部分动画过慢（0.2秒/帧）
   - 攻击动画缺乏冲击感

---

## 🚀 解决方案

### 1. 精灵图来源选择

**选择标准**：
- ✅ 开源/CC授权（避免版权问题）
- ✅ 高质量（清晰的角色细节）
- ✅ 完整动画（包含格斗必需动画）
- ✅ 适合中国武术主题（或通用格斗）

**最终选择**：
- 来源：OpenGameArt.org
- 精灵图：Chibi Grunt Fighter（超萌格斗角色）
- 授权：CC0（公共域，无需署名）
- URL：https://opengameart.org/content/chibi-grunt-base-several-fighting-styles-1
- 作者：Puffolotti

### 2. 精灵图规格

**技术规格**：
```
文件大小: 19KB (原3.4MB已压缩)
图像尺寸: 512x512像素
布局: 8行 × 8列 = 64帧
单帧尺寸: 64x64像素
颜色模式: RGBA (支持透明)
格式: PNG
```

**动画布局**：
```
第0行: Idle/站立 (8帧)
第1行: Walk/行走 (8帧)
第2行: Run/奔跑 (8帧) [未使用]
第3行: Jump/跳跃 (8帧)
第4行: Attack1/攻击1 (8帧) → 映射为轻攻击
第5行: Attack2/攻击2 (8帧) → 映射为重攻击
第6行: Hurt/受伤 (8帧)
第7行: Death/死亡 (8帧) [未使用]
```

### 3. 代码调整

**修改文件**: `src/sprite_system.py`

**修改内容**：
```python
# 旧动画定义（不统一，帧数少）
animations_def = {
    "idle": (0, 4, 0.2),
    "walk": (1, 6, 0.1),
    "jump": (2, 4, 0.15),
    "attack_light": (3, 4, 0.1),
    "attack_heavy": (4, 6, 0.12),
    "hit": (5, 3, 0.1),
}

# 新动画定义（统一8帧，速度优化）
animations_def = {
    "idle": (0, 8, 0.15),         # 更自然的呼吸感
    "walk": (1, 8, 0.08),         # 更流畅的步伐
    "jump": (3, 8, 0.12),         # 完整跳跃弧线
    "attack_light": (4, 8, 0.08), # 快速出拳
    "attack_heavy": (5, 8, 0.1),  # 重击蓄力+收招
    "hit": (6, 8, 0.08),          # 受击硬直
}
```

**关键改进**：
1. 所有动画统一为8帧（更流畅）
2. 优化帧持续时间（更符合格斗节奏）
3. 跳跃动画使用第3行（原精灵图第2行是奔跑）

### 4. 文件替换

**操作步骤**：
```bash
# 1. 下载原始精灵图
curl -L -o assets/sprites/chibi_fighter_original.png \
  "https://opengameart.org/sites/default/files/spritesheet.png"

# 2. 复制到所有角色
cp chibi_fighter_original.png default.png
cp chibi_fighter_original.png shaolin.png
cp chibi_fighter_original.png emei.png
cp chibi_fighter_original.png wudang.png
```

## 📝 后续优化建议

### 短期优化（v1.2.0）

1. **角色差异化**
   - 当前：4个角色使用同一精灵图
   - 建议：为每个角色应用调色板（palette swap）
   - 实现：修改 `sprite_system.py` 添加颜色替换
   - 效果：
     - 少林僧：橙色（金刚之力）
     - 峨眉剑客：青色（灵动之气）
     - 武当道士：绿色（道法自然）

2. **特殊招式专属动画**
   - 当前：特殊招式复用attack_heavy动画
   - 建议：使用精灵图第2行（奔跑）或第7行（死亡）作为特殊招式
   - 映射：
     - 少林"金刚身"：第7行（爆气效果）
     - 峨眉"疾风步"：第2行（冲刺动作）
     - 武当"太极气盾"：第7行（蓄力动作）

### 中期优化（v1.3.0）

3. **寻找武术主题精灵图**
   - 当前：通用Chibi格斗风格
   - 目标：中国武术特色精灵（太极/剑术/拳法姿势）
   - 来源：
     - OpenGameArt.org（搜索"martial arts", "kung fu"）
     - itch.io（付费/免费武术资源包）
     - 自制：基于当前精灵图修改添加武术元素

4. **增加动画种类**
   - 防御动画（独立于idle）
   - 胜利动画（KO后展示）
   - 倒地动画（完整death动画）

### 长期优化（v2.0.0）

5. **更高分辨率**
   - 当前：64x64像素/帧
   - 目标：128x128或256x256像素/帧
   - 需求：寻找高分辨率格斗精灵图资源

6. **多层精灵系统**
   - 身体层（基础角色）
   - 服装层（武术服饰）
   - 武器层（剑/棍/扇）
   - 特效层（气功/光效）
   - 参考：LPC (Liberated Pixel Cup) 分层系统
